１　制作物のカテゴリー：Webサイト

２　制作物の概要説明

　架空の銀行「二郎銀行」のWebサイトを作成しました。ロゴは半沢直樹風ロゴジェネレーターというWebサービスがあったため、それを使って作成し、GIMPで雑にくり抜いています。
本Webページ構成は以下のようになっています。

  • ログイン

    • ログインフォームが表示されます。簡易的な銀行機能の実装を計画していましたが、余裕がなかったため、取りやめました。

  • 口座開設

    • 実在する銀行のWebページを参考に作成しました。開設フォームにはたどり着けますが、開設はできません。

  • 金融商品

    • 架空の金融商品を３種類用意して紹介しています。画像も3種類用意しており、自動で切り替わります。フェードアウトはしません。クリックすることで、各金融商品の説明部分へ移動できます。画像自体は私が過去に撮影した画像をもとにIncscapeで作成しています。イラレはお金がもったいないので使っていません。

  • お問い合わせ

    • お問い合わせページへのリンクはページのフッター部分に配置しました。何故なら大方のサイトがお問い合わせを分かりにくい場所へ隠す傾向があるからです。私もそれに習いました。

本Webページは途中までレスポンシブデザインを意識して作っていますが、時間が掛かって仕方がなかったため、一部スマホ中心のデザインとなっています。PC向けも意識していた名残で、アコーディオンメニューは画面を狭めないと表示されないためご注意下さい。

３　相互評価する受講者に特に見てもらいたい注目点、および上手く作り込めなかった反省点

　アコーディオンメニューにはかなりこだわっています。メニューが横から出てくるタイプをはじめに考えたのですが、項目数以上に画面を占有してしまうため、上から必要な分だけ表示されるようにしました。また、高速でクリックを繰り返してもレイアウトが崩れないようにあの手この手で対策をしています。メニューの展開にはアニメーションを付けているため、必要な領域が用意されてから遅れてメニューが表示されるのもポイントです。メニューの三本線はクリックすると変形しますが、真ん中の線は不要になるため、ゆっくり消して、残った2本の位置を調整するめんどくさい処理を施しています。画面のサイズを変更する際にメニューが展開しているとレイアウトが崩れてしまったため、画面サイズの変更を検知して、メニューを自動で閉じるようにしています。バグではありません。

４−２　HTMLとCSSを使いこなしているか

  • HTML

    • HTMLはほぼすべてPHPファイルに記述したため、PHPファイルの数と行数を書きます。
    ファイル数は８つで、行数はVimで見る限り合計で346行です。

  • CSS

    • CSSファイルは画面サイズに合わせて3種類用意し、共通のCSSも用意しました。アコーディオンメニューだけ別でCSSファイルを作ったため、合計で5つのファイルがあります。行数はかぶっているところもありますが、合計で591行です。なお、山内先生の教科書に載っていたリセットCSSを写経して使用していますが、これは写しただけなのでカウントに含めていません。

    ４−３　JavaScriptとjQueryとAjaxをどのように使用したか

    • JavaScript

      • JavaScriptは、金融商品の画像切替に使用しました。imgタグのsrcを取得し、switch case で次に表示する画像を設定しています。この処理をsetIntervalを用いて5秒ごとに実行することで自動で切り替わるようになっています。

    • jQuery

      • 主にアコーディオンメニューで使用しました。メニューアイコンの変形やメニューの展開に使用しています。cssプロパティの参照と変更に多用しました。行数にして23行です。
      他にもフォームにバリデーションをかけるためjQuery Validation Plugin を利用しています。行数にして、99行です。
      おしゃれなポップアップアラートを表示するためにSweetAlertというプラグインも利用しました。記述は2行のみです。

    • Ajax

      • トップページの下部に為替レートを少しだけ表示しており、そこで使用しています。為替レートは欧州中央銀行のAPIから取得しており、AjaxによりJSON形式で取得しています。クリックするとAPIから為替レートを引っ張ってくるようになっています。該当部分の行数は3行でexchangerate.jsにその他必要な処理もまとめて記載しています。

    ４−４　PHPを使いこなしているか

    ヘッダーとフッターは各ページで共通となるため、それぞれ別ファイルに分けて、各ページで共通のヘッダーとフッターを読み込むようにしました。この処理にPHPを用いています。
    口座開設ページではGETパラメータで返すHTMLを変化させるためにPHPを使用しています。
    問い合わせフォームではPOSTパラメータによる動作の変化やクッキーの値の表示でPHPを使用しています。

    ４−５　Cookieを使いこなしているか

    問い合わせページからの問い合わせは一人1回に制限しているため、過去に問い合わせ済みかどうかをクッキーを用いて判定しています。またクッキーは問い合わせの受付番号を記録する目的にも使用しています。
